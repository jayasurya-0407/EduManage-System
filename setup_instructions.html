👨‍💼 Admin Login Not Working? Follow These Steps
1 Start XAMPP & Verify Services
  • Open XAMPP Control Panel (usually in taskbar or Start menu)
  • Click Start for both Apache and MySQL
  • Wait for both to show "Running" status (green)
  • ✓ If they start → Continue to Step 2
  • ✗ If MySQL won't start → Check if port 3307 is available, or change port in my.ini
2 Initialize the Database
  • Click Here to Run Setup
  • You should see green messages confirming:
    • ✓ Created admins table
    • ✓ Inserted default admin OR already exists
  • If you see errors → Go to Step 4 for diagnostics
3 Test the Login
  • Open Login Page
  • Use these credentials:
    • Role: Select "Admin"
    • Email: admin@example.com
    • Password: admin123
  • Click Log In
  • ✓ Should redirect to Admin Dashboard
  • ✗ If still not working → Go to Step 4
4 Run Diagnostics (If Previous Steps Don't Work)
  • Open Debug Tool
  • This tool will check:
    • ✓ Database connection status
    • ✓ All tables in database
    • ✓ Admin records count
    • ✓ Password verification test
    • ✓ Test login form
  • Look for ✓ or ✗ icons to identify what's wrong
  • Follow the recommendations in the red text
5 If Still Having Issues - Check These
  • Database Port Mismatch:
    • Open db.php file in editor
    • Check port in: mysql:host=127.0.0.1;port=3307
    • Go to XAMPP → MySQL → Config to see actual port
    • Update port in db.php if different
  • Clear Browser Cache & Cookies:
    • Press Ctrl+Shift+Delete in browser
    • Delete cookies for localhost
    • Close browser completely and reopen
  • Check PHP Errors:
    • Look in XAMPP folder: apache/logs/error.log
    • Or MySQL logs folder
🔧 Troubleshooting Quick Reference
Problem Likely Cause Solution
Database connection failed MySQL not running or port wrong Start MySQL in XAMPP, check port in db.php
Setup script shows blank page PHP error or database issue Run debug_login.php for details
No admin records found Database not initialized Run setup_admin_table.php
Login shows "Invalid credentials" Wrong email/password or corrupt hash Try admin@example.com / admin123, or run setup again
Login works but redirects to login again Session not persisting Clear cookies, check session save path, restart browser
Stuck on blank page after login File permission or PHP error Check browser console (F12 → Console) for errors
